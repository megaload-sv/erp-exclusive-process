# PR-018 — Semantic Query Engine

## Capability

C-012 introduces the first semantic query language for the TraceOps Semantic Runtime.

## Delivered

- `RuntimeQuery`
- `RuntimeQueryResult`
- `RuntimeKernelInterface::query()`
- entity selectors for components, properties, types, capabilities, metadata and slots
- identity, name and attribute filters
- relationship-aware filters for capabilities and properties
- ordering, limits, first and count operations
- iterable and countable result collections
- Runtime health registration
- unit tests
- Runtime documentation

## Example

```php
$components = $kernel
    ->query()
    ->components()
    ->supporting('clickable')
    ->havingProperty('label')
    ->orderBy('name')
    ->get();
```

## Architectural boundary

The Query Engine depends on `RuntimeKernelInterface` and accesses semantic knowledge through the Kernel. Consumers do not coordinate specialized registries.

## Base and head

- Base: `feature/runtime-kernel`
- Head: `feature/semantic-query-engine`
